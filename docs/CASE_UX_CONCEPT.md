# CORK Case UX Concept

## Status
Продуктовый документ. Без реализации. Без миграций. Без изменения кода.

## Цель
Зафиксировать продуктовую рамку для будущих фаз CORK: карточка дела, verdict bar, силы реакции, аргументы, страница дела, онбординг культуры.

---

## 1. Продуктовый принцип

CORK — суд над claims на крутость.

CORK не против денег, вещей или красивой жизни.
CORK против попытки выдать потребление, бренд, картинку или fake flex за реальную ценность.

**Коронуют:**
- skill
- труд
- пользу
- смелость
- оригинальность
- proof
- реальное дело

**Клоунят:**
- пустой понт
- overhype
- fake flex
- отсутствие proof
- потребление как «величие»

**Правило:**
> Клоуним claim, не человека.

## 2. Карточка дела в ленте (Feed Card)

Карточка должна быть понятна за 2 секунды.

### Обязательные элементы

| Элемент | Пример |
|---|---|
| Заявка | «Я сделал open-source библиотеку для X» |
| Короткий контекст | категория, дата, автор |
| Verdict bar | визуальное перетягивание 👑/🤡 |
| Verdict label | «Пока: 👑 Корона» |
| Counts | 👑 42 · 🤡 19 · 💬 18 аргументов |
| CTA | [👑 Корона] [🤡 Клоун] |
| Ссылка | [Открыть дело] |

### Важно

Не использовать в ленте сложные тексты:
- ❌ «спорно, ближе к короне»
- ❌ «71 веса»
- ❌ «42 судьи»
- ❌ «коэффициент уверенности»

Лента — простой взгляд. Детали — внутри дела.

### Wireframe (маркдаун)

```
[Заявка: «Я сделал open-source библиотеку для X»]
[Контекст: IT · 2026 · @vasya]

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
👑 64% ━━━━━━━━━━━━━━━━ 🤡 36%
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Пока: 👑 Корона

👑 42   🤡 19   💬 18 аргументов

[👑 Корона]  [🤡 Клоун]  [Открыть дело]
```

## 3. Verdict Bar Concept

Verdict bar — главный символ суда. Показывает перетягивание 👑/🤡.

### Правила

- Bar показывает только визуальное перетягивание.
- Внутри bar не должно быть перегруза текстом.
- Проценты можно показывать рядом/над bar, но не лепить всё внутрь.
- В compact-карточках (профиль) bar должен быть чистым — только сегменты.

### 4 состояния

#### 1. Crown leading
```
👑 73% ━━━━━━━━━━━━━━━━━━━━━━━━ 🤡 27%
```
Пока: 👑 Корона

#### 2. Clown leading
```
👑 31% ━━━━━━━━ 🤡 69%
```
Пока: 🤡 Клоун

#### 3. Close / спорное
```
👑 52% ━━━━━━━━━━━━━━━━ 🤡 48%
```
Пока: ⚖️ Спорно

#### 4. No verdict
```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Нет вердикта
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## 4. Tiered Reactions — Простой UX

### Принцип

- Обычный вердикт доступен широко.
- Сильные реакции ограничены.
- Влияние нельзя покупать деньгами.

### Поток

1. Пользователь нажимает 👑 или 🤡.
2. Сразу ставится обычный вердикт.
3. После этого появляется компактный блок:

```
Хочешь сильнее?
[обычный] [сильный] [мега]
```

4. Сильный/мега тратят ограниченные **charges**.
5. Charges **НЕ** покупаются за деньги напрямую.

### Уровни

| Уровень | Значок | Сила | Описание |
|---|---|---|---|
| 1 | 👑 | x1 | Обычная корона |
| 2 | 👑🔥 | x2 | Сильная корона |
| 3 | 👑💎 | x3 | Мега-корона |
| 1 | 🤡 | x1 | Обычный клоун |
| 2 | 🤡🔥 | x2 | Жёсткий клоун |
| 3 | 🤡💣 | x3 | Мега-клоун |

### Почему это важно

- Дает возможность выразить сильную уверенность.
- Не создаёт множественных реакций — один вердикт, разная сила.
- Усиливает экономику бюджета.
- Ограниченные charges предотвращают pay-to-win.

### Что НЕ проектируем сейчас

- Финальная экономика charges (сколько, как часто, от чего зависят).
- Покупка charges за деньги.
- Автоматическое начисление charges.

## 5. Arguments Concept

### Аргументы — не обычные комментарии

Комментарии в CORK — не чат. Это аргументы суда.

### На карточке в ленте

- Не просто «Комментарии».
- Лучше «Аргументы».
- Показывать 1–2 preview аргумента, если есть место:
  - 👑 «..."
  - 🤡 «..."

### На странице дела

Группировка по стороне:

- 👑 За корону
- 🤡 За клоуна
- ⚖️ Вопросы / нейтрально
- 🔗 Пруфы — возможно позже

### Типы аргументов

| Тип | Значок | Описание |
|---|---|---|
| 👑 Аргумент за корону | корона | Почему это реально круто |
| 🤡 Аргумент за клоуна | клоун | Почему это понт/fake |
| ❓ Запрос пруфа | вопрос | Требование доказательства |
| 💬 Общее | нейтрально | Обычное обсуждение |

### Будущая механика (не в MVP)

- «Сильный аргумент» — значок за лучший аргумент.
- «Не по делу» — пометка за оффтоп.
- Хороший аргумент может давать Judge Score / charges позже.
- **Сам факт написания текста НЕ усиливает голос автоматически.**
- 👍/👎 под аргументами (не в MVP).

### Почему это важно

- Превращает комментарии в структурированный суд.
- Аргументы влияют на вердикт.
- Дает материал для «Лучший аргумент» наград.

## 6. Case Page Concept

### Структура

Страница дела — арена, где всё решается.

```
[Заявка: «Я сделал open-source библиотеку для X»]
[Контекст: IT · 2026 · @vasya]

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
👑 64% ━━━━━━━━━━━━━━━━ 🤡 36%
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Пока: 👑 Корона

[👑 Корона] [🤡 Клоун]

Хочешь сильнее?
[обычный] [сильный] [мега]

---

[Описание / пруф]

---

Аргументы:
[За корону] [За клоуна] [Вопросы]

За корону:
- 👑 argument card
  author
  maybe «Сильный аргумент» later

За клоуна:
- 🤡 argument card

Вопросы:
- ⚖️ neutral comment
```

### UX

- В ленте — только беглый взгляд.
- В деле — полный суд.
- URL: `/case/:id` или `/claim/:id`.

## 7. User Paths

### 1. Зритель

> Зашёл → увидел спорное дело → прочитал аргументы → понял тему лучше.

### 2. Судья

> Увидел claim → поставил вердикт → добавил аргумент → получил статус позже.

### 3. Скаут

> Нашёл реальное дело / пустой понт → вынес на суд → получил Scout Score.

### 4. Делатель

> Сам сделал что-то → отправил на модерацию → прошёл суд → получил архивный verdict.
